# 搜索

Glean 提供三种搜索方式，通过 `⌘K` 打开搜索面板统一访问。

## 关键词搜索（FTS）

基于 SQLite FTS5 + jieba 中文分词：

- 实时响应（< 50ms）
- 支持中英文混合
- BM25 排序 + snippet 高亮

## 语义搜索（向量）

基于本地嵌入模型（fastembed），把查询转为向量后做 KNN 检索：

- 找"意思相近"的文件，不必字面匹配
- 适合自然语言查询
- 需要 embedding 完成（首次索引后自动启动）

## 混合搜索（默认）

使用 RRF（Reciprocal Rank Fusion）融合 BM25 和向量结果，兼顾字面匹配和语义理解。

::: tip
混合搜索是默认模式，绝大多数场景下效果最好。
:::

## 使用方法

按 `⌘K` 打开搜索面板：

```
┌────────────────────────────────────────────┐
│ 🔍 搜索文件...                      ⌘K     │
├────────────────────────────────────────────┤
│ 📄 quarterly-report-2024-Q3.pdf            │
│    ...[查询词高亮]...                       │
│ 📝 融资计划.md                              │
│    ...[查询词高亮]...                       │
└────────────────────────────────────────────┘
```

- **↑↓** 选择结果
- **Enter** 打开文件
- **Tab** 切换搜索模式（混合 / 语义 / 关键词）
- **Esc** 关闭面板

## 搜索示例

| 查询 | 工作原理 |
|------|---------|
| `融资 PDF` | 关键词匹配 + 语义扩展 |
| `上周那个讲季度报告的文件` | 主要靠语义 |
| `quarterly report` | 中英文混合 |
| `.md 算法` | 扩展名过滤 + 关键词 |

## 过滤

点击搜索框右侧的过滤按钮：

- 按类型（PDF / Markdown / 代码 / 图片 等）
- 按扩展名
- 按路径（包含）
- 按修改时间
